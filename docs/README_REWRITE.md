# 新系统重构边界

## 项目目标

把旧的 Python + Qt 打卡数据处理工具，重构为纯本地桌面应用：

- `Tauri v2`
- `Rust`负责全部核心业务
- `React + TypeScript`负责界面
- 本地离线运行
- 可在 Windows 打包为`.exe`或安装包

## 明确边界

### 保留的业务能力

1. 多文件、多工作表读取
2. 三行一人的人员块解析
3. 时间规范化
4. AM/NOON 窗口判断
5. 逐人逐日聚合
6. 汇总表导出
7. 需要打卡日口径页

### 新增能力

1. 一键生成通报名单
2. 按缺勤天数筛选
3. 按缺勤次数筛选
4. `AND/OR`组合逻辑
5. 更稳健的短样本/异常表解析
6. 更严格的本地安全、路径校验与脱敏日志
7. 自动化测试矩阵、性能基线、Windows 发布说明

### 明确不做

1. 不保留 Python 运行时
2. 不桥接 Python
3. 不在前端解析 Excel
4. 不接入 HTTP、云服务、远程 API
5. 不开放不必要的 shell 权限

## 新旧系统差异原则

1. 若旧 README 与旧代码冲突，以旧代码实际行为为审计起点。
2. 若旧代码存在明显脆弱实现或隐性缺陷，不做“原样兼容”。
3. 所有关键业务口径必须先写入规格文档，再进入编码。

## 默认业务口径

1. 默认窗口预设采用旧代码实际实现：
   - `AM <= 09:11:59`
   - `NOON 11:00:00~14:11:59`
2. `14:05`在默认规则下视为有效`NOON`
3. `0`打卡但已识别姓名的人，必须进入汇总与通报名单
