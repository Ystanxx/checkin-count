# 执行计划

本次重构按`codex_tauri_rust_react_attendance_tasks.csv`的顺序执行，
严格覆盖`T001~T017`。总体策略是：先冻结规格和风险边界，再搭建
纯`Tauri v2 + Rust + React + TypeScript`工程，再逐步落地解析、
聚合、通报名单、导出、测试、性能、安全和打包。

## 范围

- 在范围内：
  - 旧 Python 项目行为审阅与规则冻结
  - 纯 Rust 核心链路重建
  - React 前端页面与状态管理
  - Tauri command、最小权限与本地离线运行
  - 测试矩阵、性能基线、Windows 打包配置
  - 最终验收清单与迁移报告
- 不在范围内：
  - 保留 Python 运行时或桥接 Python
  - 接入远程 API、HTTP、云服务
  - 为兼容旧缺陷而照搬脆弱实现

## 任务顺序

1. `T001`：审阅旧代码，冻结业务规则、旧实现歧义、重构边界。
2. `T002`：初始化纯 Tauri v2 + Rust + React + TypeScript 工程。
3. `T003`：确定分层架构、DTO、错误边界、线程模型。
4. `T004`：建立领域模型、规则模型和统一错误类型。
5. `T005`：实现 Excel 读取层，支持多文件、多 sheet、坏文件隔离。
6. `T006`：实现稳健的“三行一人”块解析器，去除`>=5`硬门槛依赖。
7. `T007`：实现时间规范化与窗口判定，并把窗口改为规则注入。
8. `T008`：实现聚合器，确保`0`打卡但已识别姓名的人进入汇总。
9. `T009`：实现“一键生成通报名单”，支持天数/次数与`AND/OR`。
10. `T010`：实现 Rust 原生导出`xlsx/csv`与通报名单导出。
11. `T011`：实现 Tauri command 与后台任务、进度事件、输入校验。
12. `T012`：实现 React 页面、预览 Tabs、日志面板、分页/虚拟滚动。
13. `T013`：建立 benchmark 与性能基线，记录优化结果。
14. `T014`：完成最小权限、安全、路径与日志脱敏。
15. `T015`：建立测试矩阵、fixture、单元测试、集成测试、导出冒烟。
16. `T016`：完成 Windows release 配置、产物说明与发布文档。
17. `T017`：输出验收清单与最终迁移报告，核对`T001~T017`落地情况。

## 并行策略

- 可并行：
  - 文档冻结阶段的旧代码审阅与环境盘点
  - Rust 领域层与前端类型层
  - 导出层与通报名单过滤层
  - 测试样本准备与前端页面搭建
- 串行：
  - 规则冻结`->`领域建模`->`解析/聚合`->`导出/命令接线`
  - 安全权限收敛放在 command 设计完成后统一审计
  - 打包与验收必须在测试矩阵和权限审计之后执行
