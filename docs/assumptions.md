   绝对路径文件视为同一任务源，本次执行以工作区副本为准。

## 规格冻结决策

1. **窗口默认口径**
   - 旧 README 写的是：
     - `AM <= 09:04:59`
     - `NOON 11:00:00~14:04:59`
   - 旧代码`app/processor.py`实际实现的是：
     - `AM <= 09:11:59`
     - `NOON 11:00:00~14:11:59`
   - 新系统决策：
     - 窗口必须来自规则配置，而不是散落硬编码
     - 默认预设采用**旧代码实际行为**
     - 因此默认情况下，`14:05`计入有效`NOON`
2. **0 打卡人员**
   - 新系统必须保证：只要姓名已被识别出来，即使没有任何有效打卡，
     也必须进入汇总、缺勤计算和通报名单链路。
3. **块解析策略**
   - 不再使用“识别到`>=5`个日期列才算日期行”的硬门槛。
   - 新系统采用评分式探测，兼容短样本、异常表、夹杂备注行的情况。
4. **日志与路径**
   - 默认日志不输出完整绝对路径和完整个人明细。
   - 用户界面展示相对可读信息；内部调试信息单独分层。

## 环境假设

1. `node v22.17.0`与`pnpm 10.13.1`可用。
2. 当前机器**未发现 Rust 工具链**：
   - `cargo`不可用
   - `rustup`不可用
   - `~/.cargo/bin`不存在
3. `conda`可用，`use`环境可进入，但旧项目自检依赖
   `openpyxl`缺失，因此旧自检未能完整运行。
4. 当前网络能力受限，不能假设可在线下载依赖。

## 落地策略

1. 即使缺少 Rust 编译环境，仍继续完成：
   - 规格冻结
   - 工程目录重构
   - Rust/Tauri/React 代码文件落地
   - 测试样本与文档生成
2. 与编译、运行、打包强绑定的验证项，如果受 Rust 环境阻塞，
   将在`docs/migration-report.md`、`docs/release.md`、
   `docs/acceptance-checklist.md`中明确标注。
3. 文档统一放在`docs/`下，包含任务清单中提到的重写 README 文档。

## 非目标

1. 不保留 Python 运行时作为新系统的一部分。
2. 不让前端直接解析 Excel、聚合数据或写导出文件。
3. 不启用 shell、HTTP、任意文件系统等不必要权限。
